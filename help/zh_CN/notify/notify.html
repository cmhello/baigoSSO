系统会在“通知接口 URL”上附加 m=notify 参数，以便接收通知的系统识别。系统通知与同步登录通知的区别是：系统通知是由 SSO 服务器直接发起，而同步登录通知则通常是根据同步登录接口返回的结果，由访问者浏览器发起。

 

通知公共参数

通知公共参数是指向所有接口发起通知时都传递的参数。

名称 类型 必须 描述
app_id int true 应用的 APP ID，后台创建应用时生成的 ID。详情查看 应用。
app_key string true 应用的 APP KEY，后台创建应用时生成的 KEY。详情查看 应用。
time int true Unix 时间戳，baigo SSO 允许 +-30 分钟以内的时差，为了防止时区设置不同导致的时差，请开发者将应用的时区设置为为与 baigo SSO 一致，关于时区设置，请查看 系统设置。
sign string true 签名，将通知参数中的加密参数解码，得到一个 JSON 对象，直接将此对象签名加以签名，签名方式请查看 签名。

通知测试

接口说明

本接口用于测试通知接口是否正常。

HTTP 请求方式

GET

通知参数是否加密

否

 

通知参数

名称 类型 描述
a string 通知动作，值为 test。
echostr string 输出字符串，直接将该字符串输出便可确认该应用的通知接口正常。
 
通知参数示例

PHP 通过 print_r($_GET);

array(
    "a"           => "test", //通知动作
    "app_id"      => 1, //APP ID
    "app_key"     => "sdferi84hkdfufdsERTeugroe7treie", //APP KEY
    "time"        => 1446527788, //Unix 时间戳
    "echostr"     => "i84hksdfuw98&*Ijiiwer" //输出字符串
    "sign"        => "sdfdsfsdrewerwugroe7treie", //签名
);
